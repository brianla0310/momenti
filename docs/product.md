## 0. 지금 버전의 핵심

**Momenti는 커피 기록 앱이 아니다.**  
Momenti는 **사진을 스티커로 만들고, 하루에 붙이고, 도시에서 떠오르는 귀여운 순간을 발견하는 앱**이다.

제품의 중심축은 세 가지다.

| 축 | 역할 | 한 줄 설명 |
|---|---|---|
| **Sticker Diary** | 매일 쓰는 이유 | 내 사진을 누끼 스티커로 만들고 다이어리에 붙인다. |
| **Live Bubble Map** | 도시를 여는 이유 | 카페/젤라또 장소 위에 1시간짜리 버블이 떠서 지금 분위기를 보여준다. |
| **Artist Drops** | 장기 수익화/콘텐츠 공급 | 선정 작가가 만든 스티커팩을 Beans로 구매한다. |

핵심 원칙:

> **만드는 건 자유. 공개는 제한. 판매는 심사.**

## 1. 한 줄 정의

**Momenti — 사진을 떼어 붙이는 스티커 다이어리 + 지금 도시의 분위기를 보여주는 라이브 버블 맵.**

카페와 젤라또는 Momenti의 이탈리아 감성과 지도 경험을 묶어주는 핵심 앵커다. 하지만 제품의 중심은 음료 로그나 패스포트가 아니라 **내 사진을 누끼 스티커로 만들고, 한국식 다이어리 꾸미기 감성으로 하루를 남기고, 1시간짜리 버블로 도시의 현재 분위기를 보는 것**이다.

### 포지셔닝 문장

> **Peel your moments. Stick them in your diary. See the city bubble.**

### 더 날카로운 한국어 정의

> **사진을 스티커로 만들고, 하루에 붙이고, 도시에서 떠오르는 귀여운 순간을 발견하는 앱.**

### 부제 후보

| 후보 | 용도 |
|---|---|
| **Sticker Diary & Live Map** | 앱스토어/웹사이트 기본 부제 |
| **Caffè, Gelato, Stickers** | 이탈리아 감성 캠페인 |
| **Peel, Stick, Puff** | 온보딩/브랜드 영상/소셜 카피 |
| **Stickers Around You** | 지도 기능을 전면에 둘 때 |

금지/후순위 언어:

- `Coffee Passport`는 메인 부제에서 사용하지 않는다.
- `passport`, `brew calendar`, `coffee log` 중심 문법은 Brewwww!류 앱과 유사하게 보이므로 전면 카피에서 제외한다.
- 도장/수집 요소는 보조 재미로 남기되, 제품 정체성으로 올리지 않는다.

## 2. 타겟 & 런칭 웨지

### 주요 타겟

| 세그먼트 | 니즈 | Momenti가 주는 가치 |
|---|---|---|
| **한국식 다꾸/저널링 감성 유저** | 사진을 귀엽게 꾸미고, 스티커로 하루를 남기고 싶음 | 누끼 스티커, 마스킹테이프, 손글씨, 콜라주 중심 모바일 다꾸 |
| **이탈리아 로컬 18–35** | 동네 카페/젤라또 일상을 가볍게 남기고 싶음 | 평가 없는 일상 기록 + 지금 동네가 살아있는 느낌 |
| **관광객** | 여행 중 카페·젤라또 순간을 귀엽게 수집하고 싶음 | 도시별 스티커, 젤라또 모드, 여행 콜라주, 지도 버블 |
| **크리에이터/일러스트레이터** | 귀여운 스티커를 팔 수 있는 새로운 채널 | 초대제 Artist Drops → 심사제 Creator Studio |
| **캐주얼 소셜 유저** | 팔로우 부담 없이 주변 분위기를 보고 싶음 | 1시간 버블, 붐빔 클라우드, 댓글/팔로워 없는 ambient 공개 |

### 런칭 웨지

초기에는 “이탈리아 전체”가 아니라 **밀라노의 특정 동네 + 다꾸 감성 유저 + 카페/젤라또 여행자**로 좁힌다.

우선 후보:

- Brera
- Navigli
- Porta Venezia
- Duomo 주변
- 디자인 학교/대학가 주변
- 한국인 관광객이 많이 찾는 젤라또/카페 동선

이유:

1. 라이브 맵은 밀도가 있어야 살아난다.
2. 다꾸 앱은 첫 사용자들의 결과물이 브랜드를 만든다.
3. 관광객은 도시별 스티커/젤라또 워크와 궁합이 좋다.
4. 밀라노에서 감성·디자인·카페·관광 동선을 동시에 테스트할 수 있다.

## 3. Aha Moment & North Star

### Aha Moment

유저는 첫 90초 안에 다음 경험을 해야 한다.

> **내 사진이 귀여운 스티커가 되어 종이 다이어리에 붙는 순간.**

첫 경험 플로우:

1. 앱 실행
2. 사진 촬영 또는 갤러리에서 사진 선택
3. 5초 안에 자동 누끼 스티커 생성
4. 흰 테두리 + 그림자 + 종이 질감 자동 적용
5. 오늘 페이지에 peel & stick
6. Moka가 첫 스티커 완성 축하
7. 이미지/짧은 peel 영상으로 공유 가능

첫날의 목표는 “장소 태깅”이 아니라 **스티커 놀이의 즉각적 재미**다. 장소 태깅, 버블, Beans, Shop은 그다음에 붙는다.

### North Star Metric

**Weekly Sticker Pages Created**  
주간 기준 저장된 다이어리 페이지 수.

이 지표를 선택하는 이유:

- 단순 가입/방문보다 실제 제품 가치를 잘 반영한다.
- 스티커 제작, 다이어리 꾸미기, 리텐션, 공유 가능성이 모두 들어 있다.
- 스티커팩 구매와도 자연스럽게 연결된다.

### 핵심 보조 지표

| 단계 | 지표 |
|---|---|
| Activation | 첫 세션 내 첫 스티커 생성률 |
| Activation | 첫 스티커를 다이어리에 붙인 비율 |
| Activation | 첫 페이지 저장률 |
| Creation | 유저당 주간 생성 스티커 수 |
| Creation | 페이지당 부착 스티커 수 |
| Retention | 24시간 내 두 번째 스티커 제작률 |
| Social | 페이지 공유율 |
| Social | 버블 게시율 |
| Map | 지도 오픈율 / Moka 발견율 |
| Monetization | 스티커팩 구매 전환율 |
| Monetization | Beans 충전 전환율 |

## 4. 핵심 원칙 — 안 하는 것이 차별화다

1. **커피 기록 앱이 아니다.** 커피·젤라또는 장소 앵커이고, 제품의 본질은 스티커 다이어리다.
2. **스티커 제작은 자유.** 셀카, 친구 사진, 풍경, 음식, 영수증, 반려동물, 밈처럼 개인 다이어리 안에서 쓰는 스티커 제작에는 카테고리 제한을 두지 않는다.
3. **공개 레이어는 큐레이션.** 지도 버블·공개 페이지·장소 도장에는 제한을 둔다. 자유 제작 스티커가 곧바로 공적 지도에 올라가면 안 된다.
4. **판매 레이어는 심사.** 판매용 스티커/스탬프는 카카오톡 이모티콘처럼 신청·검토·IP 확인·품질 심사를 거친다.
5. **별점·리뷰·맛 프로필 없음.** 하트는 감정 표현이지 평가가 아니다.
6. **영구 리뷰 DB 없음.** 장소 평판을 쌓는 서비스가 아니라, 도시의 순간 분위기를 보여주는 서비스다.
7. **팔로우·팔로워 수 없음.** 소셜은 관계 그래프가 아니라 ambient discovery다.
8. **댓글 기본 제외.** 초기에는 스티커 리액션, 저장, peel count 같은 가벼운 반응만 실험한다.
9. **정확한 실시간 개인 위치 공개 금지.** 버블은 사람에게 붙는 것이 아니라 장소에 붙는다.
10. 결과물은 통계가 아니라 **간직할 물건**이다. 콜라주, 스티커 시트, 월간 다이어리, Moka Letter가 핵심 결과물이다.

## 5. 제품 구조

### 메인 탭

| 탭 | 목적 | 핵심 기능 |
|---|---|---|
| **Diary** | 내 다꾸 공간 | 월간/일간 페이지, 스티커 부착, 콜라주, 페이지 공유 |
| **Studio** | 스티커 제작소 | 사진 누끼, 테두리, 그림자, 텍스트, 스티커팩 관리, export |
| **Map** | 지금 도시 분위기 | 1시간 버블, 붐빔 클라우드, Moka Trails, 장소 태깅 |
| **Shop** | 꾸미기 재료 | 기본팩, 시즌팩, Artist Drops, Moka 코스튬 |
| **Me** | 내 자산/설정 | Beans, 내 스티커팩, 공개 링크, 동기화, 개인정보 |

Studio는 독립 탭으로 둔다. 그래야 Momenti가 “기록 앱”이 아니라 **스티커를 만들고 꾸미는 앱**으로 인식된다.

### 코어 루프

사진 찍기/가져오기 → 자동 누끼 → 스티커로 저장 → 다이어리에 peel & stick → 카페/젤라또면 장소 태깅 → 선택적으로 1시간 버블 공유 → Beans/스티커/Moka 보상 → 다시 꾸미기

### 브랜드 룰

> **un’ora e puff 🫧**  
> 버블은 1시간 뒤 사라진다. 기록은 내 다이어리에 남고, 도시의 분위기는 흘러간다.

## 6. 로드맵 & 기능 스펙

### v1 — Sticker Diary MVP

목표: 첫날부터 혼자 써도 재밌는 “다꾸 앱”으로 완결한다.

- **스티커 다이어리 캘린더**: 월간 뷰 + 일간 페이지 + 자유 배치.
- **Studio**: 사진 → 누끼 스티커. iOS VisionKit / Android ML Kit Subject Segmentation 우선 검토. 필요 시 U2NetP 폴백.
- **스티커 제작 자유 + Stickerbook 제한**: 셀카·풍경·음식·반려동물·영수증·밈 등 개인 다이어리용 카테고리 제한 없음. 단 무료 버전은 `내가 생성한 스티커` 활성 보관함을 100개로 제한한다.
- **Magic Border**: 누끼 후 자동 흰 테두리, 그림자, 종이 질감 적용.
- **Tape & Label Kit**: 마스킹테이프, 날짜 라벨, 젤라또 영수증 라벨, 손글씨 캡션.
- **Page Templates**: “오늘의 카페”, “젤라또 3칸”, “주말 산책”, “여행 하루” 템플릿.
- **Stickerbook / 내 스티커 목록**: 내가 만든 스티커를 보관·검색·팩으로 묶기. 무료 버전은 개인 생성 스티커 100개 활성 슬롯을 기본값으로 둔다.
- **Peel back to Stickerbook**: 다이어리에 배치한 스티커를 떼어 다시 스티커북으로 돌려보내기. 새 스티커를 중복 생성하지 않고 재사용하게 만드는 UX로 정의한다.
- **첫 90초 온보딩**: 샘플 사진 또는 내 사진으로 첫 스티커 제작 → 즉시 다이어리에 붙이기.
- **기본 데코팩**: 무료 기본팩 + 시즌 맛보기팩.
- **Beans 기초 루프**: 데일리 기록, 첫 스티커 제작, 첫 카페/젤라또 태깅, Moka 튜토리얼 보상.
- **카페/젤라또 태깅**: Places 연동. 태깅은 선택형.
- **카페·젤라또 도장**: 보조 수집 요소. 도장/패스포트 문법을 메인 카피로 쓰지 않는다.
- **Share Page**: 오늘의 페이지를 이미지로 공유.
- **Peel Recording**: 스티커가 뜯기고 붙는 과정을 짧은 영상으로 export하는 기능을 v1 실험 후보로 둔다.
- **WhatsApp export 실험**: 내가 만든 개인 스티커팩을 “Export Pack / Add to WhatsApp” 형태로 내보내기. 기존 WhatsApp 스티커 import는 v1 범위에서 제외.
- **계정 없이 시작**: 로컬 저장 우선. 동기화·기기 이전은 추후 계정 옵션으로 제공.

### v1.1 — Moka Trails / Sticker Walks

목표: 지도 콜드스타트를 해결하고, 유저 버블이 적어도 지도에 볼거리를 만든다.

- **Moka Trails**: Moka가 만든 도시별 스티커 산책 코스.
- **Sticker Walks**: Brera, Navigli, Porta Venezia 같은 동네별 카페/젤라또 스티커 코스.
- **Geo-unlock Stickers**: 특정 동네/장소 근처에서만 얻는 스티커 조각.
- **City Starter Pack**: 밀라노 시작 유저에게 무료 도시팩 제공.
- **Weekend Drop**: 주말 한정 Moka 출몰/스티커 드롭.
- **Creator Walk 후보**: 작가가 만든 스티커 코스를 지도에 배치하는 실험.

핵심: 라이브 맵이 처음부터 “사람이 많은 지도”일 필요는 없다. **지도 자체가 스티커를 얻는 놀이터**가 되어야 한다.

### v1.2 — Live Bubble Map Beta

목표: “지금 여기가 핫하네”를 평점이 아니라 분위기로 보여준다.

- **1시간 버블 맵**: 카페/젤라테리아 장소 위에만 버블 표시.
- **버블 내용 제한**: 카페/젤라또 기록 카드 + 한줄 감정 + 공개 허용된 데코만.
- **셀카/친구 얼굴 기본 제외**: 자유 제작 스티커는 기본적으로 버블에 올리지 않는다.
- **☁️ 지금 붐빔 클라우드**: 1시간 내 버블 3개 이상이면 장소 위에 구름/스팀/반짝임 표시.
- **Mood Weather**: 사람 수가 아니라 분위기 날씨를 보여준다. 예: `gelato cloud`, `espresso rush`, `sunny terrace`, `quiet corner`.
- **Moka 지도 출몰**: Moka가 지도 위에 랜덤 등장. 발견하면 Beans, 한정 스티커, 시즌 미션 진행.
- **Map AR 우선**: 카메라 AR이 아니라 살아있는 지도 레이어. 2D/3D 버블, 스팀, 구름, Moka 애니메이션으로 AR 느낌을 만든다.
- **위치 보호**: 사용자 현재 위치 노출 없음. 장소 단위 태그만 공개. 백그라운드 위치 추적 금지.
- **신고/숨기기**: 버블 신고, 사용자 차단, 특정 장소 버블 숨기기.

### v1.5 — Link-based Social Diary

목표: 인스타식 피드가 아니라 “남의 다꾸 구경”을 만든다.

- **Public Page by Link**: 전체 계정 공개가 아니라 특정 페이지/월간 페이지를 링크로 공유.
- **월간 다이어리 공개**: 초기에는 링크 공유 중심. 지도에서 남의 전체 프로필로 깊게 유입시키지 않는다.
- **팔로우 없음**: follow/follower 카운트 없음. 구경하기, 저장하기, 가벼운 리액션만.
- **댓글 기본 제외**: 댓글은 넣지 않는다. 스티커 리액션/peel count 정도만 실험.
- **얼굴 포함 경고**: 공개 전 “사람 얼굴이 포함되어 있을 수 있음” 경고.
- **공개 전 미리보기**: 어떤 스티커와 장소 정보가 공개되는지 명확히 보여준다.
- **Sticker Swap 후보**: 친구에게 내가 만든 스티커를 보내고, 친구가 자기 다이어리에 붙일 수 있게 하는 기능. 판매/재배포는 불가.
- **Moka Letters**: 월말 리캡을 Moka가 편지처럼 전달. 예: “이번 달 너는 pistacchio 색을 7번 썼고, gelato 순간을 4번 붙였어.”

### v2 — Artist Drops / Creator Studio

목표: Momenti를 스티커 소비/창작 경제권으로 확장한다.

초반 명칭은 **Market**보다 **Artist Drops**가 우선이다. 공급이 적어 보이는 마켓보다, “이번 주 드롭 뭐 나왔지?”라는 기대감을 만든다.

단계:

| 단계 | 명칭 | 내용 |
|---|---|---|
| 1 | **Moka Drops** | 자체 제작 Moka/시즌팩 |
| 2 | **Artist Drops** | 초대 작가 5–10명 한정 출시 |
| 3 | **City Drops** | 밀라노/로마/피렌체 도시팩 |
| 4 | **Creator Studio** | 신청제 심사 마켓 |
| 5 | **Brand Collab** | 카페/젤라또/문구 브랜드 콜라보 |

기능:

- **선정/인증 크리에이터제**: 초반에는 초대제. 이후 신청제.
- **스티커/스탬프팩 판매**: 귀여운 스티커, 도시팩, 카페팩, 젤라또팩, Moka 콜라보, 시즌팩.
- **심사 프로세스**: 신청 → 포트폴리오 검토 → 규격 체크 → IP/상표/초상권 검토 → 품질 심사 → 출시.
- **Beans 결제**: 유저는 Beans로 구매. 내부적으로 paid Beans / bonus Beans 출처를 분리 관리.
- **정산**: 스토어 수수료 및 환불/차지백 차감 후 순수익 기준 크리에이터 8 : 회사 2를 기본안으로 둔다.
- **크리에이터 페이지**: 작가 소개, 판매팩, 출시 예정 드롭. 팔로워 수 경쟁은 만들지 않는다.
- **Export 권한**: 유료 크리에이터팩은 기본적으로 WhatsApp export 금지. 작가가 opt-in한 “export allowed” 팩만 외부 export 허용.

### v2.5+ — 확장 후보

- **실물 다꾸 키트**: 대량 수익 사업이 아니라 작은 브랜드 증거로 먼저 실험. 첫 100명 베타 유저에게 Moka 실물 스티커 우편 발송, 카페 QR + Moka 스티커 배치, Artist Drop 작가와 한정 실물 시트 제작.
- **Moka Table AR**: 카페 테이블 위에 Moka를 앉혀 사진을 찍고 다이어리에 붙이는 안전한 카메라 AR 후보.
- **카메라 AR 지도**: 지도 레이어가 충분히 검증된 후 선택적으로 검토. v1/v1.2에서는 하지 않는다.
- **도시 확장**: 밀라노 → 로마 → 피렌체 → 베네치아/나폴리 등 관광축.
- **브랜드 콜라보**: 카페 B2B는 매우 보수적으로. 광고성 추천/평점 오염 금지.

## 7. 콘텐츠 레이어 설계

핵심 원칙: **제작은 자유, 공개는 제한, 판매는 심사.**

| 레이어 | 허용 범위 | 검수/제한 | 예시 |
|---|---|---|---|
| **Private Diary** | 거의 자유 | 공개 전까지 최소 개입 | 셀카, 친구, 파스타, 영수증, 풍경, 밈, 반려동물 |
| **Public Bubble** | 강한 제한 | 자동 필터 + 신고 | 카페/젤라또 장소 버블, 한줄 감정, 공개 허용 스티커 |
| **Public Page** | 중간 제한 | 이미지/텍스트 필터 + 신고 | 링크로 공유한 일간/월간 페이지, 여행 콜라주 |
| **Artist Drops / Creator Studio** | 가장 강한 제한 | 사전 심사 + IP 검증 | 판매용 스티커팩, 스탬프팩, 시즌팩, 브랜드 콜라보 |

### Private Diary

- 개인 다이어리에 넣는 스티커는 자유 제작.
- 얼굴, 친구 사진, 음식, 풍경도 허용.
- 비공개 콘텐츠는 랭킹·검색·지도 노출 없음.
- 클라우드 동기화가 들어가면 개인정보 처리방침과 삭제/내보내기 기능 필요.

### Public Bubble / Public Page

- 공개 전 사용자가 정확히 무엇이 보이는지 확인해야 한다.
- 얼굴·민감 이미지가 포함될 가능성이 있으면 경고.
- 버블은 장소 단위로만 노출. 실시간 개인 위치 금지.
- 계정 전체 공개보다 **페이지 단위 링크 공유**가 기본이다.
- 신고, 숨기기, 차단, 반복 위반 제한 기능 필수.

### Artist Drops / Creator Studio

- 판매 스티커는 전수 심사.
- 저작권/상표권/초상권 침해 금지.
- 선정된 작가만 판매 가능. 일반 유저는 개인 스티커 제작은 가능하지만 판매는 불가.
- 판매팩에는 사용 범위와 export 가능 여부를 명시한다.

## 8. 스티커 스튜디오 / 외부 스티커 연동

### 스티커 제작 기능

| 기능 | 설명 |
|---|---|
| **Auto Cutout** | 사진에서 인물/사물 자동 누끼 |
| **Manual Refine** | 지우개/복원 브러시로 누끼 수정 |
| **Magic Border** | 자동 흰 테두리, 그림자, 종이 질감 |
| **Tape & Label Kit** | 마스킹테이프, 날짜 라벨, 영수증 라벨 |
| **Tiny Preview Test** | 지도 버블/작은 스티커에서도 잘 보이는지 미리보기 |
| **Sticker Pack Builder** | “내 밀라노 여행팩”, “젤라또 투어팩”처럼 묶기 |
| **Peel Recording** | 스티커를 떼어 붙이는 과정을 짧은 영상으로 export |
| **Moka Auto Decor** | Moka가 빈 공간에 어울리는 테이프/도장 추천 |
| **Page Templates** | “오늘의 카페”, “젤라또 3칸”, “주말 여행” 템플릿 |

### Stickerbook / 내 스티커 목록

목표: 무료 제한은 “창작 자유”를 막는 장벽이 아니라, **정리된 스티커북과 Pro 업그레이드 이유**가 되어야 한다.

#### 데이터 모델 원칙

| 개념 | 정의 | 제한 적용 | 예시 |
|---|---|---|---|
| **Sticker Asset** | 사용자가 생성한 원본 스티커 자산 | 무료 100개 제한 적용 | 누끼 딴 셀카 스티커, 젤라또 스티커 |
| **Sticker Instance** | 다이어리 페이지 위에 배치된 개별 사본 | 제한 미적용 | 같은 젤라또 스티커를 월요일/토요일 페이지에 각각 붙임 |
| **Sticker Pack** | 여러 asset을 묶은 컬렉션 | pack 자체는 제한 미적용, 내부 asset은 제한 적용 | 내 밀라노 여행팩 |
| **Flattened Page Copy** | 공유/보관용으로 렌더링된 페이지 이미지 | 제한 미적용 | 삭제된 스티커가 붙어 있던 과거 페이지 이미지 |

#### 무료 버전 제한

- 무료 버전은 **개인 생성 Sticker Asset 100개 활성 슬롯**을 기본 제한으로 둔다.
- 제한은 “스티커를 몇 번 붙였는가”가 아니라 “재사용 가능한 개인 스티커 자산을 몇 개 보관하는가”에 적용한다.
- 기본팩, Moka 무료팩, Artist/Creator 구매팩은 개인 생성 100개 제한에 포함하지 않는다.
- import한 외부 이미지/공유 시트 이미지로 만든 스티커는 개인 생성 asset으로 계산한다.
- 같은 asset은 여러 페이지에 자유롭게 배치할 수 있다. 배치 횟수 제한은 두지 않는다.

#### Peel back to Stickerbook

- 다이어리에 붙인 스티커를 길게 누르거나 peel gesture로 떼면 **Remove from page / Return to Stickerbook / Duplicate / Delete** 선택지를 제공한다.
- `Return to Stickerbook`은 새 asset을 생성하지 않는다. 이미 존재하는 asset reference를 유지하고, 해당 page instance만 제거한다.
- 이 기능의 목적은 “갯수 제한 우회”라기보다 **중복 생성 방지 + 실물 다꾸 감성**이다. 유저가 같은 사진을 계속 새로 누끼 따지 않게 만든다.
- 삭제는 신중하게 처리한다. `Delete from Stickerbook`을 해도 과거 다이어리 페이지는 깨지지 않도록, 페이지에는 flattened copy 또는 snapshot을 남긴다. 단 삭제된 asset은 새 페이지에 다시 붙일 수 없다.

#### Pro / 수익화 연결

- 무료: 개인 생성 스티커 활성 슬롯 100개.
- Momenti Pro 후보: 개인 생성 스티커 활성 슬롯 확장 또는 무제한, 클라우드 백업, 고해상도 export, 고급 정리 기능.
- 단순히 “더 만들려면 결제”로 보이면 반감이 크다. 문구는 **“더 큰 Stickerbook”**, **“스티커 보관함 확장”**, **“여행/시즌별 무제한 컬렉션”**에 가깝게 간다.
- Beans로 임시 슬롯을 파는 방식은 복잡해질 수 있으므로 초기에는 Pro 또는 일회성 `Stickerbook Expansion` IAP 중 하나만 실험한다.

### Import

초기 import는 WhatsApp이 아니라 일반 파일/사진 기반으로 간다.

- 사진/이미지 파일 import.
- iOS/Android 공유 시트로 가져오기.
- 클립보드 이미지 붙여넣기 검토.
- 기존 WhatsApp 스티커팩을 Momenti로 직접 가져오는 기능은 공식 가능성과 권리 이슈가 불명확하므로 v1에서 제외.

### Export

- **내가 만든 개인 스티커팩**: WhatsApp export / Add to WhatsApp 실험.
- **Momenti 무료 바이럴팩**: export 허용 가능.
- **Creator 유료팩**: 기본 export 금지.
- **Export allowed Creator Pack**: 작가가 동의한 경우에만 별도 가격/권한으로 허용.

### WhatsApp export의 역할

WhatsApp export는 핵심 제품이 아니라 **바이럴 기능**이다.

권장 플로우:

> Momenti에서 개인 스티커팩 제작 → WhatsApp에 추가 → 친구가 “이 스티커 뭐야?”라고 물음 → Momenti 설치

자연스러운 출처 표시:

- pack name: `Made with Momenti`
- tray icon: 작은 Moka
- 개인 이미지 자체에는 강제 워터마크를 넣지 않는다.

### 라이선스 원칙

구매자는 Momenti 앱 안에서 개인 다이어리, 콜라주, 공유 이미지에 사용할 수 있다. 재판매, 재배포, 별도 스티커팩 추출, 상업적 사용은 금지한다. 외부 메신저 export는 해당 pack이 “export allowed”일 때만 가능하다.

### Creator Pack 라이선스 3단계

| 라이선스 | 사용 가능 범위 | 가격 전략 |
|---|---|---|
| **In-app only** | Momenti 다이어리 안에서만 사용 | 기본가 |
| **Share allowed** | 다이어리 이미지/영상 export에 포함 가능 | 기본 포함 권장 |
| **Messenger export allowed** | WhatsApp 등 외부 스티커팩으로 내보내기 가능 | 별도 가격/작가 opt-in |

중요: 유저가 산 스티커를 자기 콜라주 이미지에 넣어 Instagram에 올릴 수는 있어야 한다. 원본 스티커 파일을 외부 메신저팩으로 빼내는 것은 별도 권한으로 둔다.

## 9. Live Bubble Map / AR 지도

### 지도 철학

Momenti의 지도는 리뷰 지도가 아니다. **기분 지도**다.

- “여기 평점 4.8”이 아니라 “지금 여기서 뭔가 귀여운 일이 일어나는 중”.
- 장소 추천보다 분위기 관찰.
- 영구 리뷰보다 1시간짜리 도시의 맥박.

권장 표현:

> “지금 여기 핫해”보다 **“지금 여기 귀여운 순간들이 떠 있어.”**

### 감정 태그

| 태그 | 의미 |
|---|---|
| `cozy` | 조용하고 따뜻함 |
| `sweet` | 젤라또/디저트 무드 |
| `sunny` | 테라스/날씨 좋음 |
| `busy but cute` | 붐비지만 분위기 좋음 |
| `moka was here` | Moka 출몰 |
| `tiny joy` | 작은 행복 |

피해야 할 평가성 표현:

- `best`
- `worst`
- `overrated`
- `must visit`
- `tourist trap`
- 별점/순위/랭킹 언어

### 버블 규칙

- 지속시간: **1시간 고정**.
- 위치: 카페/젤라테리아 장소 단위.
- 내용: 공개 가능한 기록 카드 + 한줄 감정.
- 히스토리: 1시간 뒤 지도에서 사라짐. 내 다이어리에는 남음.
- 프로필 링크: 기본 없음. public page가 켜진 유저도 버블에서 깊은 프로필 유입을 과도하게 만들지 않는다.
- 노출 로직: 가까운 곳, 최근, 장소 밀도, 신고 품질을 기준으로 가볍게 정렬.

### 지금 붐빔 클라우드

| 조건 | 표현 |
|---|---|
| 버블 1개 | 작은 말풍선 하나 |
| 버블 3개 이상 | 구름/스팀 이펙트, `sta succedendo qualcosa` |
| 버블 8개 이상 | Moka 출몰 확률 증가, 한정 Beans/스티커 드롭 가능 |
| 1시간 경과 | puff 효과 후 사라짐 |

### Moka Trails / Sticker Walks

지도 콜드스타트 해결용 핵심 기능이다.

예시:

- `Milano Gelato Sticker Walk`
- `Brera Caffè Morning`
- `Navigli Night Gelato`
- `Porta Venezia Tiny Joy Trail`

원칙:

- “여기가 최고”가 아니라 “이 동네를 걸으면 이런 스티커를 모을 수 있다”.
- Moka가 안내하는 산책 코스처럼 보이게 한다.
- 유저 버블이 적어도 지도에 스티커 조각, 코스, Moka 이벤트가 있어야 한다.

### Google Maps / 실시간 데이터 구현 전략

결론: **Google Maps는 장소 DB와 지도 렌더링 기반으로 쓰고, “실시간으로 여기가 뜨겁다”는 데이터는 Momenti의 1st-party 버블 이벤트로 만든다.**

Google Maps Platform의 역할:

| 용도 | 추천 API/기술 | Momenti에서의 쓰임 |
|---|---|---|
| 장소 검색/태깅 | Places API / Places SDK | 카페·젤라테리아 검색, Place ID 확보, 기본 장소 정보 |
| 지도 표시 | Maps SDK / Maps JavaScript API | 2D 지도 위에 Momenti 버블·Moka·스팀 오버레이 |
| 몰입형 지도 | Map Tiles API / Photorealistic 3D Tiles | 후순위: 3D 도시 지도 위에 버블을 띄우는 고급 Map AR |
| 카메라 AR | ARCore Geospatial API | 후순위: 실제 거리에서 장소 근처 버블/Moka를 앵커링 |

중요한 제한:

- Google Maps 앱의 “인기 시간대/실시간 붐빔” 데이터를 그대로 가져오는 공식 공개 API에 의존하지 않는다. 공식 Places API는 장소 검색, 상세 정보, 사진, 영업시간, 리뷰/평점 등 장소 정보를 제공하는 용도로 본다.
- Google Maps/Places 데이터를 스크래핑하지 않는다. Places/Maps 콘텐츠는 캐싱·저장 제한과 attribution 요구사항이 있으므로, Momenti DB에는 Google Place ID와 Momenti 자체 이벤트를 중심으로 저장한다.
- Place ID는 장기 저장 가능한 장소 키로 사용하되, 장소명/사진/리뷰/평점 같은 Google 제공 콘텐츠는 정책에 맞게 요청·표시·갱신한다.

Momenti 실시간 데이터 모델:

| 테이블/스트림 | 저장 내용 | TTL/보관 | 목적 |
|---|---|---|---|
| **PlaceIndex** | `place_id`, 카테고리, 대략 좌표, Momenti 내부 태그 | 장기. Google 정책에 맞게 refresh | 카페/젤라또 장소 앵커 |
| **BubbleEvent** | `place_id`, mood, sticker preview, created_at, expires_at, abuse flags | 공개 노출 1시간. 운영 로그는 최소화 | 지도 위 1시간 버블 |
| **HeatAggregate** | 5/15/60분 rolling count, unique user count, mood mix | 짧은 TTL | 지금 붐빔 클라우드 계산 |
| **MokaDrop** | 위치/동네/시간대별 Moka 출몰 이벤트 | 캠페인 기간 | 콜드스타트와 재방문 유도 |

실시간 점수 예시:

```text
heat_score(place) = Σ time_decay(bubble_age)
                  + unique_user_bonus
                  + moka_drop_boost
                  - spam_risk_penalty
```

표시 원칙:

- 개인의 현재 위치는 표시하지 않는다. 장소 단위 버블만 표시한다.
- `지금 붐빔 클라우드`는 최소 3명 또는 3개 이상의 독립 버블부터 표시한다.
- 유저 밀도가 낮은 초기에는 Google Places의 정적 장소 정보보다 **Moka Trails, City Starter Pack, Weekend Drop**으로 지도의 빈 느낌을 줄인다.
- “real-time AR map”의 real-time은 Google에서 import한 군중 데이터가 아니라 Momenti 내부의 1시간 버블 이벤트에서 만든다.

구현 단계:

1. **v1.2 Map AR**: Google Map 또는 유사 지도 위에 Momenti 버블/스팀/Moka 오버레이. 카메라 AR 없음.
2. **v1.3 Realtime Overlay**: WebSocket/Firebase/Supabase Realtime 등으로 현재 viewport의 BubbleEvent와 HeatAggregate만 구독.
3. **v2 3D Map AR 후보**: Photorealistic 3D Tiles 또는 커스텀 3D 지도 렌더러로 지도 몰입감 강화. 비용과 성능 검증 후 진행.
4. **v2.5 Camera AR 후보**: ARCore Geospatial로 장소 근처에 Moka/버블을 앵커링. 안전·권한·배터리·VPS 커버리지 검증 후 진행.

### AR 지도 원칙

- v1/v1.2에서는 카메라 AR을 하지 않는다.
- “AR 느낌”은 지도 위 2D/3D 레이어로 구현한다.
- 배터리, 카메라 권한, 실공간 안전 문제를 피한다.
- 후순위로 카메라 AR을 검토하더라도 사람 얼굴/민감 장소/사유지 앵커링 이슈를 먼저 해결해야 한다.
- 카메라 AR을 한다면 공개 위치 앵커 AR보다 **Moka Table AR**이 우선이다.

## 10. 스코프 결정

**제품 정체성 = 스티커 다이어리 + 라이브 버블 맵.**  
**지도/도장/획득 앵커 = 카페 ☕ + 젤라또 🍦.**

- 카페·젤라또는 Momenti의 이탈리아 감성, 여행 SEO, 장소 기반 버블을 묶는 핵심 2종.
- 풍선·장소 도장·지도 핫스팟은 이 2종만 우선 지원.
- 개인 다이어리 스티커의 **카테고리는 제한하지 않는다.** 파스타, 아페리티보, 셀카, 풍경도 개인 페이지에는 자유롭게 붙일 수 있다. 단 무료 버전의 개인 생성 스티커 활성 보관함은 100개를 기본 제한으로 둔다.
- **아페리티보 🍹 = 로드맵 후보**: 저녁 시간대 사용성을 확보할 필요가 생기면 검토.
- **식사 카테고리화는 보류**: 전 음식 확장 = 음식일기 앱들과 경쟁 + 브랜드 희석.
- 좁은 지도 앵커와 넓은 개인 스티커 제작을 분리한다. 이 분리가 Momenti의 자유도와 브랜드 선명도를 동시에 만든다.

## 11. 디자인 시스템

### 11.1 디자인 한 줄 방향

**진짜 종이 다이어리를 스마트폰 안에 넣은 듯한, 귀엽지만 너무 유아적이지 않은 스티커 소셜 맵.**

핵심 조합:

> **Korean diary deco × Italian bar/gelato culture × tactile sticker UI**

Momenti의 디자인은 “예쁜 앱”이 아니라 제품 해자다. 기능만 보면 스티커 앱, 다이어리 앱, 지도 앱은 각각 존재한다. 그러나 Momenti는 **스티커를 실제로 떼어 붙이는 촉감 + 한국식 다꾸 + 이탈리아 로컬 감성 + 1시간 버블 지도**를 하나의 화면 언어로 묶어야 한다.

톤 기준:

- 귀엽지만 부끄럽지 않다.
- 아기자기하지만 촌스럽지 않다.
- 장난스럽지만 게임 UI처럼 과하지 않다.
- 종이와 스티커의 물성이 느껴지지만 사용성은 명확해야 한다.

### 11.2 디자인 판단 기준 5가지

| 키워드 | 의미 | 적용 예시 |
|---|---|---|
| **Tactile** | 손으로 만지는 느낌 | 종이, 테이프, 스티커, 잉크, 뜯김, 햅틱 |
| **Collected** | 하루의 조각이 쌓이는 느낌 | 다이어리 페이지, 스티커북, 월간 콜라주 |
| **Ambient** | 시끄러운 SNS가 아니라 주변 분위기 | 1시간 버블, mood cloud, 팔로워 없는 구경 |
| **Playful** | 작고 귀여운 놀이성 | Moka, Beans, puff, 스티커 드롭 |
| **Local** | 이탈리아 동네 감성 | bar al banco, gelato, 영수증, 도시별 trail |

피해야 할 디자인 방향:

| 피해야 할 느낌 | 이유 |
|---|---|
| **Fintech/SaaS clean UI** | Momenti의 물성·감성과 맞지 않음 |
| **Instagram feed UI** | 팔로우/좋아요 경쟁처럼 보임 |
| **게임 UI 과다** | Beans와 Moka가 P2E/가챠처럼 보일 위험 |
| **너무 어린이 앱 같은 귀여움** | 20–30대 다꾸 유저에게 유치해 보일 수 있음 |
| **여행 패스포트/우표 과다** | Brewwww!류 passport/calendar 문법과 다시 가까워짐 |

### 11.3 시각 세계관 — 종이 위의 도시

Momenti의 화면 전체는 **종이 다이어리 위에 도시 조각이 붙어 있는 느낌**이어야 한다.

| 화면 | 디자인 은유 | 핵심 시각 요소 |
|---|---|---|
| **Diary** | 월간 다이어리 페이지 | dot grid, 날짜 라벨, 붙은 스티커, 빨간 펜 표시 |
| **Studio** | 책상 위 스티커 작업대 | 사진 조각, 가위/누끼 점선, Magic Border, 테이프 |
| **Map** | 종이 지도 위에 떠 있는 버블 | 흐린 지도, 버블, mood cloud, Moka, trail path |
| **Stickerbook** | 투명 필름 스티커 보관 파일 | 100개 카운터, 필름 질감, 카테고리 탭 |
| **Shop / Drops** | 문구점 신상품 코너 | 스티커팩 봉투, 한정 드롭, Artist cards |
| **Moka** | 다이어리 위를 돌아다니는 고양이 | 온보딩, 보상, 빈 화면, 지도 출몰, 드롭 안내 |

중요 원칙:

- 앱 UI 자체는 정돈되어 있어야 한다.
- 사용자의 다이어리 페이지 안에서만 자유롭게 지저분해져도 된다.
- 카드 UI를 쓰더라도 “앱 카드”가 아니라 “종이 조각”처럼 보여야 한다.

### 11.4 팔레트

기본 정체성은 **cream + espresso + bialetti red**다. caffè/gelato 모드는 테마로 존재하지만, 앱 전체의 중심 톤은 따뜻한 종이와 잉크다.

| 역할 | 컬러 | 용도 |
|---|---|---|
| **Paper Base** | `#FFF8EA` / `#FFFBF2` | 카드, 다이어리 페이지, 스티커북 필름 배경 |
| **Cream Background** | `#F8EFDD` | 앱 전체 배경 |
| **Espresso Ink** | `#33211A` | 주요 텍스트, 지도 라벨, 아이콘 |
| **Bialetti Red** | `#C8331B` | 핵심 CTA, 기록 버튼, 오늘 표시 |
| **Pistacchio** | `#7C9A4E` | 성공, 자연, 젤라또 보조 |
| **Azzurro** | `#6FA8C9` | 지도, 거리, 하늘, 물 |
| **Amarena Pink** | `#E85D8A` | 젤라또/시즌/귀여운 강조 |
| **Tape Beige** | warm translucent beige | 마스킹테이프, 라벨 |

운영 원칙:

- 한 화면에서 강한 포인트 컬러는 1–2개만 쓴다.
- 파스텔은 배경/장식에 쓰고, 본문 텍스트는 진한 espresso 계열로 유지한다.
- Gelato 화면은 fragola/amarena/menta를 쓸 수 있지만, 텍스트 대비를 희생하지 않는다.
- 접근성 기준으로 일반 텍스트는 최소 4.5:1, 큰 텍스트는 3:1 대비를 목표로 한다.

### 11.5 타이포그래피

현재 방향인 **Fredoka + Nunito** 조합은 유지한다.

| 용도 | 폰트 방향 |
|---|---|
| 앱 로고 / 큰 제목 | Fredoka 700 |
| 섹션 제목 | Fredoka 600 |
| 버튼 / 탭 / 라벨 | Nunito 700 또는 800 |
| 본문 / 안내 | Nunito 600 |
| 캡션 | Nunito 700, small |
| 스티커팩 이름 / 짧은 감정 문구 | Fredoka 또는 제한적 손글씨풍 보조 폰트 |

손글씨 폰트는 UI 전체에 쓰지 않는다. 손글씨는 **스티커, 라벨, 짧은 감정 문구**에만 제한한다.

### 11.6 레이아웃 원칙 — 카드 UI보다 페이지 UI

1. **모든 화면은 종이 페이지 위에 구성한다.**
   - flat white 대신 paper/cream 배경.
   - 약한 dot grid, ruled line, paper grain 사용.

2. **정렬은 유지하되 스티커는 살짝 비대칭이다.**
   - 스티커는 -8도~+8도 정도 랜덤 회전.
   - 테이프도 살짝 각도를 달리한다.
   - 단, 텍스트 영역과 주요 CTA는 흔들지 않는다.

3. **중요 CTA는 문구점 스티커처럼 보인다.**
   - `+` 버튼은 일반 floating action button보다 큰 원형 스티커.
   - 누르면 peel/pop 느낌을 준다.

4. **지도는 유틸 지도보다 종이 지도다.**
   - Google Maps를 그대로 보여주면 장소 유틸 앱처럼 보인다.
   - 실제 지도 위에는 Momenti식 버블/스티커/Moka 레이어를 덮는다.

### 11.7 화면별 디자인 방향

#### Diary

Diary는 앱의 홈이다. 느낌은 **“이번 달 내 스티커 페이지”**다.

추천 요소:

- 월간 캘린더는 얇은 dot grid 종이 위에 배치.
- 날짜마다 붙은 스티커는 살짝 회전.
- 오늘 날짜는 빨간 펜 동그라미 느낌.
- 빈 날짜는 흐릿한 dotted placeholder.
- 하단에는 “오늘 붙인 조각들” 또는 “오늘의 페이지” 진입.
- 월말에는 자동 콜라주/Moka Letter로 변환.

문구 예시:

```text
Luglio 2026
23 stickers pasted · 5 pages saved

Oggi
붙일 순간이 기다리고 있어요
```

기존의 `총 커피 수`, `스트릭`, `패스포트 진행률` 같은 데이터 앱 느낌은 낮추고, **붙인 스티커 수 / 저장한 페이지 / 오늘의 페이지** 중심으로 표현한다.

#### Studio

Studio는 Momenti의 핵심 화면이다. 느낌은 **“사진을 스티커로 만드는 작업대”**다.

기본 플로우:

```text
사진 선택
→ 누끼 진행: 가위/마법 점선 애니메이션
→ Magic Border 적용
→ 테이프/라벨/펜/스티커 장식
→ Diary에 붙이기 / Stickerbook에 저장
```

보완 디테일:

- 누끼 완료 시 “툭” 하고 흰 테두리가 생긴다.
- 스티커 아래에 따뜻한 그림자가 생긴다.
- 손잡이 점으로 resize/rotate 가능.
- Magic Border 옵션:
  - White cutout
  - Soft shadow
  - Paper edge
  - Receipt label
  - Gelato gloss
- 저장 버튼 문구는 `Save`보다 `Stickerbook에 넣기`, `오늘 페이지에 붙이기`를 우선한다.

#### Map

Map은 “실시간 핫플 지도”가 아니라 **“도시 위에 귀여운 순간들이 떠 있는 기분 지도”**다.

추천 요소:

- 실제 지도 위에 단순 핀만 놓지 않는다.
- 버블은 말풍선보다 비눗방울/젤리 스티커 느낌.
- 사람이 많으면 `🔥 hot`보다 `☁️ little cloud`, `🫧 bubbling now`.
- 버블이 사라질 때 `puff` 애니메이션.
- 장소명은 리뷰 앱처럼 크게 노출하지 않고 작게 둔다.
- Moka는 지도 위에서 낮잠, 걷기, 콩 줍기 등 랜덤 행동.

| 상태 | 표현 |
|---|---|
| 버블 1개 | 작은 투명 버블 |
| 버블 3개 이상 | 스팀/구름/mood cloud |
| 버블 8개 이상 | Moka 등장 또는 한정 drop hint |
| 사라지기 직전 | 작아지고 흐려짐 |
| 인기 장소 | 랭킹 배지 금지, 분위기 이펙트만 |

#### Stickerbook

Stickerbook은 Passaporto를 대체하는 화면이다. 느낌은 **“내가 만든 스티커 보관 파일”**이다.

추천 요소:

- 상단: `My Stickerbook`
- 카운터: `72 / 100 stickers`
- 카테고리:
  - Created by me
  - Moka
  - Caffè
  - Gelato
  - Artist Drops
  - Seasonal
- 스티커는 투명 필름 위에 붙어 있는 느낌.
- 비활성/삭제된 스티커는 흐릿한 자국만 남김.
- 유료 확장 유도는 강압적이지 않고 “스티커북이 통통해졌다”는 은유로 표현.

문구 예시:

```text
72 / 100 stickers
Moka says: your stickerbook is getting chubby 🫘
```

무료 제한은 벌칙이 아니라 **스티커북이 꽉 찬 물성 있는 상황**으로 보여준다.

#### Shop / Drops

초기에는 `Shop`보다 **Drops**라는 감각이 더 좋다. 느낌은 “상점”보다 **문구점 신상품 코너**다.

추천 구조:

- `This week’s drop`
- `Moka Picks`
- `Milano Pack`
- `Artist Drop`
- `Coming back soon`
- `Owned`

상품카드는 결제 UI보다 작은 봉투/스티커팩 패키지처럼 보이게 한다.

```text
Ferragosto Pack
🌞 12 stickers · limited drop
20 Beans
```

### 11.8 Moka 디자인 방향

Moka는 단순 캐릭터가 아니라 **제품 운영자**다. 온보딩, 지도, 보상, 빈 화면, 드롭 안내를 이끈다.

비주얼 원칙:

- 몸통: 둥근 고양이.
- 귀: 모카포트 뚜껑처럼 보이는 실루엣.
- 꼬리: 커피 증기처럼 말림.
- 얼굴: 점눈 + 작은 입.
- 컬러: espresso brown + cream face + red scarf/accent.
- 흰 스티커 테두리는 기본 적용.
- 실루엣만 봐도 알아볼 수 있어야 한다.
- 3D, 복잡한 그라데이션, 과한 질감 금지.

초기 기본 포즈:

| 포즈 | 사용처 |
|---|---|
| 기본 Moka | 온보딩, 빈 화면 |
| Sleepy Moka | 지도 콜드스타트 |
| Bean Moka | Beans 보상 |
| Gelato Moka | 젤라또 모드 |
| Artist Moka | Studio/스티커 제작 안내 |
| Tiny Moka | 지도 위 랜덤 출몰 |

처음부터 많은 코스튬을 만들지 않는다. **기본 6포즈**로 제품 안에서 반복 노출하고, 시즌 코스튬은 라이브옵스 자산으로 쌓는다.

### 11.9 스티커 스타일 가이드

| 요소 | 규칙 |
|---|---|
| 테두리 | 기본 흰색 3–6px |
| 그림자 | 아래쪽으로 아주 약한 warm shadow |
| 회전 | -8도 ~ +8도 |
| 최소 크기 | 손가락으로 조작 가능한 크기 유지 |
| 배경 | 투명 PNG/WebP 또는 canvas asset |
| 텍스트 포함 | 작은 화면에서 읽히는지 검사 |
| 유료팩 | 패키지 썸네일과 실제 스티커 톤 통일 |
| 지도 버블용 스티커 | 작은 크기에서도 실루엣이 읽혀야 함 |

프로토타입에서는 CSS `drop-shadow`로 흰 누끼 테두리를 만들 수 있다. 실제 이미지 스티커에서는 canvas/SVG 렌더링으로 border를 생성하는 구조를 검토한다.

### 11.10 모션 디자인

Momenti의 브랜드는 정적인 화면보다 **움직임**에서 강해진다. 하지만 항상 움직이는 화면은 피곤하므로, 모션은 상태 변화와 피드백에만 사용한다.

시그니처 모션:

| 모션 | 느낌 | 사용처 |
|---|---|---|
| **Peel** | 스티커를 떼는 느낌 | 스티커 이동/되돌리기 |
| **Stick** | 종이에 착 붙는 느낌 | 다이어리에 부착 |
| **Puff** | 버블이 사라짐 | 1시간 버블 만료 |
| **Bob** | 살짝 떠 있음 | 버블/Moka idle |
| **Stamp** | 잉크 도장 | 장소 도장/보상 |
| **Page flip** | 종이 페이지 넘김 | 월간/일간 이동 |
| **Tear** | 테이프/라벨 뜯기 | Tape & Label Kit |

원칙:

- 기본 모션은 150–450ms 안쪽.
- Moka idle animation은 느리고 작게.
- 버블 bob은 랜덤 delay로 자연스럽게.
- 스티커 붙일 때만 강한 햅틱.
- `prefers-reduced-motion` 사용자는 bob/pop/auto motion을 최소화.

### 11.11 사운드와 햅틱

사운드/햅틱은 Momenti와 잘 맞지만, 공공장소 사용을 고려해 기본값은 조심스럽게 둔다.

| 행동 | 사운드 | 햅틱 |
|---|---|---|
| 스티커 떼기 | 작은 paper peel | light |
| 스티커 붙이기 | soft tap | medium |
| 도장 찍기 | ink stamp | medium/heavy |
| Beans 획득 | 둥근 pop | light |
| 버블 사라짐 | puff | 없음 또는 very light |
| Moka 발견 | 작은 meow/bell | light |

사운드는 기본 off 또는 매우 낮은 볼륨을 추천한다. 설정에서 켤 수 있게 한다.

### 11.12 아이콘 스타일

초기 프로토타입에서 lucide 같은 outline icon을 써도 되지만, 제품화 단계에서는 손으로 그린 듯한 둥근 outline으로 바꾼다.

| 아이콘 | 추천 방향 |
|---|---|
| Map pin | 종이 지도 핀/스티커 핀 |
| Plus | 빨간 원형 스티커 버튼 |
| Camera | 즉석카메라 + 가위 조합 |
| Stickerbook | 투명 파일북 |
| Shop/Drops | 문구점 봉투 |
| Beans | 작은 커피콩 |
| Moka | 얼굴/발자국/꼬리 실루엣 |

아이콘 선 두께는 2.25–2.75px 정도로 통일하고, 모서리는 둥글게 처리한다.

### 11.13 지도 디자인 원칙

Google Maps/Places를 쓰더라도 화면은 Google Maps 앱처럼 보이면 안 된다. Momenti 지도는 실제 지도 위의 **Momenti layer**가 핵심이다.

```text
Base map: 정보량을 줄이고 흐리게 표시
Momenti layer:
- 장소 스티커 핀
- 버블
- mood cloud
- Moka
- trail path
- sticker unlock marker
```

지도에서 하지 말 것:

- 별점 표시.
- 리뷰 개수 표시.
- 가격대 표시.
- `Top 10` 랭킹.
- 영구 댓글.
- 정확한 유저 현재 위치 표시.

### 11.14 브랜드 보이스와 마이크로카피

핵심 액션 문구는 명확하게, 보조 문구에서만 귀엽게 간다. 너무 많은 농담은 피곤하다.

| 상황 | 일반 문구 | Momenti식 문구 |
|---|---|---|
| 첫 스티커 생성 | Sticker created | 방금 종이 위에 붙일 준비 완료 ✂️ |
| 저장 완료 | Saved | 착 붙었어요 |
| 버블 공유 | Share to map | 1시간만 도시 위에 띄우기 🫧 |
| 버블 만료 | Expired | puff… 사라졌어요 |
| 스티커북 꽉 참 | Storage full | 스티커북이 통통해졌어요 |
| 커피 오후 기록 | Saved | nonna가 봤다면 한마디 했을지도 ☕ |
| Beans 획득 | Reward earned | Moka가 콩을 주워왔어요 🫘 |

공개 지도에서는 조롱, 평가, 장소 비난처럼 보이는 문구를 피한다.

### 11.15 한국식 다꾸 요소

한국식 다꾸의 핵심은 단순히 귀여운 스티커가 아니라 **레이어링, 여백, 테이프, 라벨, 날짜, 손글씨, 작은 반복 장식**이다.

- 작은 체크박스.

- 포토카드 프레임.

## 12. Beans & 수익 모델

### 재화: Beans 🫘

Beans는 “토큰”이 아니라 **앱 내 꾸미기 재화**다. 크립토/현금화/투기성 표현을 피한다.

피해야 할 표현과 권장 표현:

| 피해야 할 표현 | 권장 표현 |
|---|---|
| token | Beans |
| earn money | collect Beans |
| cash out | creator payout |
| investment | sticker purchase |
| reward economy | decorating credits |
| P2E | 사용 금지 |

운영 원칙:

- 유저에게는 하나의 Beans 잔액처럼 보인다.
- 내부 원장에서는 **paid Beans / bonus Beans** 출처를 분리한다.
- paid Beans는 IAP/Google Play Billing으로 충전.
- bonus Beans는 활동 보상, 데일리, Moka 발견, 시즌 미션, 리워드 광고로 지급.
- paid Beans는 만료 금지.
- bonus Beans는 캠페인성 만료를 둘 수 있으나 명확히 고지.
- 일반 유저 현금화 금지.
- 확률형 유료 구매/가챠 금지.
- P2E처럼 보이는 표현 금지.

### Beans 획득

- 데일리 기록.
- 첫 스티커 제작.
- 새 카페/젤라또 태깅.
- 시즌 미션.
- Moka 발견.
- 리워드 광고: opt-in only, 하루 상한, 강제 노출 없음.

### Beans 사용

- 스티커팩.
- 시즌 한정팩.
- Artist Drops / Creator Pack.
- Moka 코스튬.
- 다이어리 테마.
- 지도 버블 이펙트.
- export allowed pack의 외부 export 권한.

### Creator Pack에서 bonus Beans 처리

크리에이터 정산 리스크를 피하기 위해 다음 원칙을 둔다.

1. Creator pack은 paid Beans 우선 소진.
2. bonus Beans 사용은 월별 프로모션 한도 내에서만 허용.
3. Momenti 공식팩/무료팩/시즌 맛보기팩은 bonus Beans 사용을 더 넓게 허용.
4. 회사가 bonus Beans로 발생한 creator sale을 인정하는 경우, 그 비용은 마케팅 비용으로 처리한다.

### 수익원 우선순위

1. **스티커팩 / 테마 / 꾸미기 재료** — 제품 본질과 가장 잘 맞음.
2. **시즌 한정 드롭** — Ferragosto, Natale, Carnevale, Sanremo, 도시별 젤라또 투어.
3. **Beans 판매** — 소액 소비 중심.
4. **Artist Drops → Creator Studio** — 장기 해자.
5. **Moka 코스튬 / 지도 버블 이펙트** — 감성 IP 수익화.
6. **Rewarded Ads** — free Beans 공급용, opt-in only.
7. **Momenti Pro** — 고급 사용자용. 너무 빨리 전면화하지 않는다.
8. **실물 다꾸 키트** — 수익원이라기보다 브랜드 증거로 작게 실험.

### Momenti Pro 후보

Pro는 스티커 무료 제공이 아니라 스튜디오/보관/공유 고급 기능 중심이어야 한다. Pro에 “모든 스티커팩 무료”를 넣으면 크리에이터 경제와 충돌한다.

- 고화질 export.
- 더 많은 백업/동기화 용량.
- 고급 테마.
- 월간 콜라주 템플릿.
- 광고 없는 bonus Beans.
- 고급 편집 도구.
- 더 긴 peel recording export.

## 13. Artist Drops / Creator Studio

### 방향

카카오톡 이모티콘처럼 누구나 소비할 수 있지만, 판매자는 선정/인증된 작가로 제한한다. 완전 오픈 마켓이 아니라 **큐레이션된 귀여움**이 Momenti의 브랜드 자산이다.

초반에는 “마켓”보다 **드롭**으로 시작한다.

- 공급이 적어도 빈약해 보이지 않는다.
- 시즌성과 FOMO를 만들 수 있다.
- 브랜드 톤을 지키기 쉽다.
- 작가를 큐레이션할 수 있다.

### 단계

| 단계 | 내용 |
|---|---|
| 1단계 | Momenti 자체 제작 기본팩 + Moka 시즌팩 |
| 2단계 | 초대제 Artist Drops: 이탈리아 로컬 작가, 한국 다꾸 작가, 여행 일러스트레이터 |
| 3단계 | City Drops: 밀라노/로마/피렌체 도시팩 |
| 4단계 | 신청제 Creator Studio |
| 5단계 | 문구 브랜드, 카페/젤라또, 캐릭터 IP 콜라보 |

### 심사 기준

- 규격: 이미지 크기, 투명 배경, 안전 영역, 썸네일.
- 품질: 가독성, 작은 크기 렌더링, 일관된 스타일.
- IP: 저작권, 상표권, 캐릭터 유사성, 팬아트 권리.
- 초상권: 실존 인물/유명인/타인 얼굴 사용 금지 원칙.
- 정책: 성적 콘텐츠, 혐오, 폭력, 괴롭힘, 불법행위, 개인정보 노출 금지.
- 브랜드 핏: 귀여움, 다꾸성, 이탈리아/여행/일상 감성과의 궁합.

### 정산 기본안

- 유저 구매 → 스토어 수수료/세금/환불/차지백 차감 → 순수익 산정.
- 순수익 기준 크리에이터 8 : 회사 2.
- 무료 Beans 프로모션으로 발생한 구매는 회사 정책에 따라 정산 인정 여부를 별도 처리.
- 정산 주기, 최소 정산액, 환불 보류 기간을 약관에 명시.

### 제한

- 일반 유저는 개인 스티커 제작 가능, 판매 불가.
- Creator pack 재판매/재배포/무단 export 금지.
- export allowed pack만 WhatsApp 등 외부 메신저 export 가능.
- 작가가 export를 허용하지 않은 유료팩은 Momenti 내부 사용만 가능.

## 14. 모더레이션 / 안전 / 개인정보

Momenti는 공개 버블, public page, creator submission을 다루기 때문에 UGC 앱으로 설계해야 한다. 모더레이션은 나중에 붙이는 기능이 아니라 v1부터 제품 구조에 들어가야 한다.

### 최소 기능

- 공개 콘텐츠 신고.
- 사용자 차단.
- 버블 숨기기.
- 반복 위반 계정 제한.
- 공개 페이지 이미지 신고.
- 텍스트 필터: 이탈리아어/영어/한국어 기본 욕설·혐오·성적 표현.
- 이미지 NSFW 필터: public page와 creator submission 우선.
- 신고 처리 큐.
- 신고 처리 로그.
- 공개 콘텐츠 삭제 플로우.

### 위치 안전

- 실시간 개인 위치 공개 금지.
- 버블은 장소 단위로만 표시.
- 백그라운드 위치 추적 금지.
- 기록 저장 시 1회성 위치/장소 태깅 우선.
- 민감 장소 주변 버블 노출 제한 검토.
- 사용자가 자신의 현재 위치와 버블 위치가 다르게 표시된다는 점을 이해할 수 있어야 한다.

### 얼굴/셀카

- Private diary에서는 셀카 스티커 허용.
- Public page/버블 공개 전 얼굴 포함 경고.
- 타인 얼굴이 포함된 콘텐츠는 신고 사유가 될 수 있음.
- Creator Studio에서는 실존 인물 얼굴 사용 금지 원칙.

### 데이터 원칙

- v1은 로컬 저장 우선.
- 계정 없이 시작 가능.
- 동기화가 들어가면 삭제/내보내기/계정 탈퇴 플로우 필수.
- 공개 범위는 기록 단위 또는 페이지 단위로 명확히 표시.
- EU 운영을 고려해 GDPR/DSA 관련 정책 검토 필요.

## 21. 코드 구현 전환 계획

### 21.1 결론

**구현 가능하다.** 다만 상용 앱 전체를 한 번에 만드는 것이 아니라, 현재 React mockup을 새 포지셔닝에 맞게 정리하고 실제 API/네이티브 기능은 service boundary로 분리한다.

구현 범위는 세 단계로 나눈다.

1. **바로 구현 가능한 프로토타입 작업**  
   React/Vite 구조화, 카피/탭 리포지셔닝, 디자인 토큰 분리, Stickerbook 100개 제한, Return to Stickerbook, localStorage, mock Live Bubble Map, Moka Trails.
2. **웹/PWA 또는 React Native에서 구현 가능한 작업**  
   사진 업로드, 간단한 누끼/크롭 플로우, 드래그/회전/리사이즈, 다이어리 페이지 export, Places 검색, 지도 오버레이.
3. **네이티브/백엔드/외부 계정이 필요한 작업**  
   iOS Vision/Android ML Kit 누끼, Google Maps/Places 키, Supabase Realtime, 신고/모더레이션 큐, IAP/Google Play Billing, WhatsApp Add to Stickers, ARCore/3D Tiles.

### 21.2 추천 개발 스택

| 영역 | 추천 |
|---|---|
| 웹 프로토타입 / 랜딩 | React + Vite + TypeScript |
| 배포 | GitHub → Cloudflare Pages |
| 모바일 MVP | Expo React Native + TypeScript |
| 백엔드 | Supabase |
| 실시간 버블 맵 | Supabase Realtime + Postgres/PostGIS |
| 이미지/스티커 저장 | Supabase Storage 또는 Cloudflare R2 |
| 지도/장소 검색 | Google Maps / Places API |
| AI 코딩 실행자 | Claude Code |

원칙:

- 지금은 React/Vite 웹 데모를 빠르게 배포한다.
- 실제 모바일 앱은 Expo React Native로 전환한다.
- FlutterFlow는 핵심 앱보다는 admin/creator submission tool에 적합하다.
- Momenti의 핵심 UX는 스티커 에디터, gesture, animation, map overlay, native cutout이므로 코드 주도 개발이 맞다.

### 21.3 GitHub / Cloudflare / Claude Code 운영 방식

권장 워크플로우:

```text
GitHub repo 생성
→ Cloudflare Pages 연결
→ main 브랜치 보호
→ Claude Code는 feature branch에서만 작업
→ PR 생성
→ Cloudflare preview URL 확인
→ 사람이 확인 후 main merge
→ production 배포
```

금지:

- Claude Code가 `main`에 직접 push하지 않는다.
- production API key를 코드에 넣지 않는다.

### 21.5 현재 프로토타입 상태

업로드된 `momenti.jsx`는 프로토타입으로는 좋은 출발점이다. 단, 새 포지셔닝에 맞춰 `Coffee Passport`, `Passaporto`, “음료 로그” 중심 언어는 제거해야 한다.

| 현재 구현 | 상태 | 다음 조치 |
|---|---|---|
| Diary calendar | 있음 | Sticker Page / 자유 배치 캔버스 쪽으로 확장 |
| Log flow | 있음 | “음료 선택”보다 “사진→스티커 생성”을 첫 경험으로 변경 |
| Map bubble | 있음 | Mood Bubble / HeatAggregate / Moka Trail mock으로 재정리 |
| Passaporto | 있음 | `Stickerbook / Me` 탭으로 교체 |
| Beans shop | 있음 | Artist Drops / Moka Drops 구조로 리네이밍 |
| Caffè/Gelato mode | 있음 | 장소 앵커로 유지하되 제품 정체성 전면에서는 낮춤 |
| emoji sticker mock | 있음 | 실제 이미지 asset 모델로 확장 |
| 앱 상태 관리 | useState 중심 | localStorage 또는 Zustand/Context로 분리 |

### 21.6 코드 구현 원칙

1. **Sticker-first**: 첫 화면과 첫 행동은 “사진을 스티커로 만들기”다.
2. **Design tokens first**: inline style과 `cp-*` 클래스를 무작정 확장하지 말고 `src/design/*`와 재사용 컴포넌트로 분리한다.
3. **Asset / Instance 분리**: 원본 스티커와 페이지 위 사본을 분리한다.
4. **로컬 우선**: v1 프로토타입은 서버 없이 localStorage/IndexedDB로 저장한다.
5. **지도는 mock → service boundary**: Google Maps/Places는 나중에 붙일 수 있도록 interface를 먼저 둔다.
6. **실시간은 내부 이벤트 기반**: Google의 실시간 붐빔 데이터가 아니라 Momenti의 BubbleEvent를 집계한다.
7. **공개 기능은 제한적으로**: Public Page by Link, Bubble opt-in, 신고 버튼을 mock이라도 포함한다.
8. **Tactile UI over generic UI**: 종이, 스티커, 테이프, Moka, 버블의 물성을 유지한다.
9. **Reduce motion support**: peel/pop/bob/puff 모션은 접근성 설정에서 줄일 수 있어야 한다.

### 21.7 우선 구현 Phase

#### Phase 0 — Design System Extraction

목표: 기능 추가 전에 Momenti의 시각 언어를 코드 구조로 고정한다.

- `cp-*` 클래스명을 `momenti-*`로 점진 변경.
- inline style에 흩어진 색, radius, shadow, motion 값을 `src/design/tokens.ts`로 이동.
- `typography.ts`, `motion.ts`, `shadows.ts`, `stickerStyles.ts` 추가.
- 공통 컴포넌트 후보:
  - `PageSurface`
  - `PaperCard`
  - `Tape`
  - `Sticker`
  - `Bubble`
  - `StickerButton`
  - `Moka`
  - `StickerbookCounter`
- `COFFEE PASSPORT`, `Passaporto`, passport cover UI 제거.
- `Stickerbook` 은유로 대체.
- Fredoka/Nunito 조합 유지.
- paper texture, tape, sticker border, reduced-motion 대응 유지.
- 백엔드/API/지도 연동은 추가하지 않는다.

#### Phase 1 — Prototype Repositioning

- 상단 카피: `COFFEE PASSPORT` 제거 → `STICKER DIARY & LIVE MAP`.
- 탭 구조: `Diary / Studio / Map / Stickerbook`.
- `LogModal` 문구: “Cosa hai davanti?”보다 “Make a sticker” 중심.
- 하트 평가 언어 축소: 맛 평가가 아니라 감정 태그/무드로 보이게 한다.
- 기존 calendar는 유지하되 “sticker page” 진입 버튼을 추가한다.

#### Phase 2 — Stickerbook Data Model

```ts
type StickerAsset = {
  id: string;
  source: 'camera' | 'gallery' | 'emoji_mock' | 'artist_pack' | 'moka_drop';
  imageUrl?: string;
  emoji?: string;
  title?: string;
  createdAt: number;
  tags: string[];
  packId?: string;
  exportPolicy: 'personal' | 'in_app_only' | 'share_allowed' | 'messenger_export_allowed';
};

type StickerInstance = {
  id: string;
  assetId: string;
  pageId: string;
  x: number;
  y: number;
  scale: number;
  rotation: number;
  zIndex: number;
};
```

무료 제한은 `StickerAsset` 개수에만 적용한다. `StickerInstance`는 제한하지 않는다.

#### Phase 3 — Peel Back to Stickerbook

- 페이지 위 스티커 길게 누르기.
- `Return to Stickerbook`: page instance만 삭제, asset은 유지.
- `Remove from page`: instance 삭제.
- `Duplicate`: 같은 asset을 새 instance로 복제.
- `Delete from Stickerbook`: asset을 비활성화하되 과거 page snapshot은 깨지지 않게 한다.

#### Phase 4 — Sticker Studio MVP

- 사진 업로드 input.
- object URL preview.
- Magic Border mock: 흰 테두리, 그림자, 종이 질감.
- 배경 제거는 초기엔 mock 또는 수동 crop으로 대체.
- `Save to Stickerbook`.
- 무료 100개 초과 시 upgrade/expand 안내.
- `Peel Recording`은 후순위. 먼저 정적 이미지 export부터.

#### Phase 5 — Live Bubble Map MVP

- BubbleEvent 타입 추가.
- `expiresAt` 기반 TTL.
- `mood` 태그: cozy, sweet, sunny, busy but cute, tiny joy.
- 장소별 rolling count로 `heat_score` 계산.
- 3개 이상이면 `Mood Cloud` 표시.
- Google 지도 대신 현재는 mock 좌표 유지.
- 나중에 Google Place ID와 연결할 수 있게 `placeId` 필드 추가.

#### Phase 6 — Moka Trails / Map Cold Start

- `MokaTrail` mock 데이터 추가.
- 예: Brera Sticker Walk, Navigli Gelato Walk, Porta Venezia Sweet Route.
- trail spot을 누르면 잠금 스티커/도시 스티커 힌트 표시.
- 지도에 Moka 출몰 이벤트 표시.
- Moka 발견 시 bonus Beans 지급 mock.

#### Phase 7 — Persistence & Export

- entries, stickerAssets, stickerInstances, beans, ownedPacks를 localStorage에 저장.
- 다이어리 페이지 이미지 export 후보: `html-to-image` 또는 `dom-to-image`.
- WhatsApp export는 실제 연동 전까지 `Export Pack` UI와 JSON/spec mock만 만든다.
