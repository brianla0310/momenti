# DESIGN.md — Momenti 디자인 원칙 (visual & interaction source of truth)

이 문서는 Momenti의 **시각·상호작용 디자인 정본**이다.
새 UI를 만들거나 기존 화면을 손대기 전에 **반드시 이 문서를 먼저 읽어라.**

- 제품 범위·로드맵·잠금 결정(D1–D15)의 정본은 [`docs/product.md`](docs/product.md) 이다.
- 이 문서(`DESIGN.md`)는 **어떻게 보이고 어떻게 만져지는가**의 정본이다.
- 둘이 충돌하면: 범위는 `product.md`, 룩앤필은 `DESIGN.md`를 따르고, 충돌 자체를 사람에게 보고하라.

> 품질 리뷰의 **일반적 방법론**(변경 전후를 스스로 검수하고, 눈에 보이는 회귀를 잡는 체크리스트 습관)은 오픈소스 [Hallmark](https://github.com/Nutlope/hallmark)에서 영감을 받았다. Hallmark의 텍스트나 규칙을 그대로 옮겨온 것은 없으며, 아래 원칙은 전부 Momenti 고유의 것이다.

---

## 1. 제품 정체성 (Product identity)

Momenti는 **사진 컷아웃 스티커 다이어리**다. 사진을 스티커로 만들고, 하루를 꾸미고, 다이어리를 책장에 책처럼 꽂아 둔다.

- 느낌: **한국식 다꾸(다이어리 꾸미기) × 촉각적인 종이/스티커 UI.** 이탈리안 바/젤라또는 **선택 가능한 하나의 테마일 뿐, 정체성이 아니다.**
- 철학: **"Analog soul, digital superpowers."** 진짜 종이의 물리·물성(뜯기·기울기·페이지 넘김)을 집요하게 흉내 내되, 디지털 초능력(무한 재사용·undo·전부 백업)을 얹는다.
- 이것은 **모바일 다꾸 경험**이지 SaaS 대시보드가 아니다. 화면은 종이 위의 오브젝트처럼 보여야 한다.

**절대 하지 말 것 (avoid-list):**
- Coffee Passport / 여권(passport) 언어, 커피·젤라또를 정체성으로 내세우기
- 별점, 영구 리뷰 DB
- 팔로워/팔로잉 수를 정체성으로 쓰기, 인스타그램식 공개 프로필 그리드
- Google Maps / 핀테크 / 유틸리티-지도 룩
- 게임 이코노미 / 화폐 중심 UI, "×N회 사용" 같은 소모성 스티커 카운트(모든 에셋은 무한 재사용)
- 가짜 질감 키치(가짜 가죽 등) — 표면 스킨이 아니라 종이의 **행동**을 흉내 낸다

## 2. 기존 시스템을 보존하라 (Preserve, don't reinvent)

Momenti는 이미 일관된 시스템 위에 서 있다. 새 작업은 이 시스템에 **합류**해야 하며, 옆에 평행 시스템을 새로 만들면 안 된다.

- **하나의 요소 모델** — 데코 표면(월간 스프레드, 하루 페이지)의 모든 요소는 [`PageElement`](apps/web/src/data/pageElements.js) 하나로 표현된다(`type: "sticker" | "text"`). 새 요소 종류(사진·테이프·낙서…)는 **새 `type` 값**으로 추가하지, 새 배열/새 상태로 만들지 않는다.
- **하나의 스티커 렌더러** — 스티커가 보이는 모든 곳은 [`StickerVisual`](apps/web/src/components/StickerVisual.jsx)을 거친다. paper/glitter 룩이 어디서나 동일해야 한다.
- **하나의 디자인 토큰 폴더** — 색·타이포·그림자·모션은 [`src/design/`](apps/web/src/design/)에서 온다. 값은 하드코딩하지 말고 토큰을 참조하라. (현재 `App.jsx`는 JSX-first 프로토타입이라 값을 인라인으로 중복 보유하지만, 토큰과 **1:1로 일치**한다 — 새 값은 토큰에서 가져와라.)
- **하나의 저장 스키마** — `localStorage` 단일 키 `momenti.v1`, 스키마 **v3**. 스키마·마이그레이션은 요청 없이 바꾸지 말고, 사용자 데이터를 삭제·이전하지 마라.

기존 시스템을 **재설계하지 마라.** 요청받지 않은 리팩터·시각 개편은 하지 않는다.

## 3. 색상 (Color)

따뜻한 종이 팔레트가 기준이다. 하드코딩 대신 [`src/design/tokens.js`](apps/web/src/design/tokens.js)의 `COLORS`를 쓴다.

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `paper` | `#FFFBF2` | 스티커/페이지 종이 흰색 |
| `cream` | `#F8EFDD` | 크레마 배경 |
| `espresso` | `#33211A` | 기본 잉크(텍스트) |
| `bialettiRed` | `#C8331B` | 모카 레드 액센트 |
| `pistacchio` | `#7C9A4E` | 피스타치오 그린 |
| `azzurro` | `#6FA8C9` | 스카이 블루 |
| `amarena` | `#E85D8A` | 아마레나 핑크(젤라또 테마) |
| `fragolaMilk` | `#FBE9F0` | 딸기라떼 배경(젤라또 테마) |

- 텍스트 잉크 팔레트는 위의 **따뜻한 5색**(espresso/bialettiRed/pistacchio/azzurro/amarena)으로 고정한다. 새로운 임의 색을 텍스트에 추가하지 마라.
- 순수 검정(`#000`)·순수 회색 UI 색은 피한다. 그림자·경계도 따뜻한 톤으로.
- 채도 높은 "형광 SaaS" 색, 파랑 링크색, 빨강 에러 배너 같은 관습적 웹 팔레트를 끌어오지 마라.

## 4. 레이아웃과 표면 (Layout & surfaces)

- **데코 표면은 종이 오브젝트다.** 월간 스프레드와 **3:4 세로 하루 페이지**가 두 데코 표면이다. 주간(week) 뷰는 **네비게이션 전용**(D2)이며 데코 표면이 아니다.
- 하루 페이지는 **3:4 비율을 유지**하며 화면 중앙에 레터박스로 놓인다. 비율을 깨서 꽉 채우지 마라.
- 모서리 반경은 토큰(`RADII`)을 따른다: 카드/시트 `20`(16–22 범위), 스티커 `16`, 알약/토글/FAB `999`.
- 그림자는 종이가 얹힌 듯한 **따뜻하고 부드러운** 그림자다(`src/design/shadows`). 딱딱한 머티리얼 엘리베이션 그림자를 쓰지 마라.
- 여백은 넉넉하게. 요소가 종이 위에서 **호흡**해야 한다. 정보 밀도로 승부하는 대시보드가 아니다.

## 5. 타이포그래피 (Typography)

폰트는 [`src/design/typography.js`](apps/web/src/design/typography.js)의 `FONTS`를 쓴다.

- **display = Fredoka** (둥근, 스티커 라벨/워드마크 느낌)
- **body = Nunito** (친근하고 읽기 쉬운 다이어리 본문)

하루 페이지 **텍스트 박스**(§D6)는 의도적으로 최소 사양이다 — **더 얹지 마라**:

- 3폰트: `hand`(Caveat, 손글씨 기본) · `round`(Fredoka) · `clean`(Nunito)
- 5잉크: §3의 따뜻한 5색
- 3크기: `S 14px` · `M 19px` · `L 26px`
- **볼드/이탤릭·정렬·링크·글머리표 없음.** 텍스트는 다꾸 손글씨 메모지, 워드 프로세서가 아니다.

## 6. 모션과 물리감 (Motion & physics)

행동이 장식보다 우선한다. 표면 스킨이 아니라 **물리**를 흉내 낸다.

- **페이지 넘김** — 모든 페이지 네비게이션에 종이 페이지-턴(`cp-pageturn`). 캘린더↔하루 페이지 진입이 대표적.
- **스티커 물리** — 놓을 때 tilt/thud/settle, 회전은 `-8°~+8°` 범위, 길게 눌러 lift → 드래그 이동 → return zone 반환(peel-back). 탭은 액션 메뉴.
- **성능 예산** — 상호작용 응답 **<100ms**, 애니메이션 **60fps** 또는 더 단순하게. 이 예산을 못 지키면 애니메이션을 **줄여라**, 프레임을 떨어뜨리지 마라.
- **`prefers-reduced-motion`을 항상 존중하라.** 감소 모션에서는 페이지-턴이 페이드로 단순화되고, 글리터 sheen은 정적으로, 그러나 **모든 기능은 그대로 동작**해야 한다.
- 모션은 **의미**가 있어야 한다(공간 관계·인과). 순수 장식 애니메이션, 통통 튀는 로더, 시차 스크롤 같은 "재미용" 모션은 넣지 마라.

## 7. 상호작용 상태 (Interaction states)

- 모든 인터랙티브 요소는 **명확한 press/active 피드백**을 준다(살짝 눌림·기울기·그림자 변화). 종이 위 물체를 만지는 느낌.
- 터치 타깃은 손가락 기준으로 넉넉히(최소 ~44px). 하루 페이지의 작은 스티커도 hold-lift로 집을 수 있어야 한다.
- 포커스 가시성·`aria-label`·역할을 지켜라. 토글/버튼을 지울 때 남은 `aria-label`·핸들러·주석까지 함께 지워라(잔재 금지).
- 파괴적/비가역 동작(삭제 등)은 명확히 구분하고, 되돌릴 수 있으면 **undo로 회수**되게 하라(하루 페이지 세션 undo/redo).
- 로딩·빈 상태도 종이 톤으로. 스켈레톤 셔머 같은 SaaS 관습을 쓰지 마라.

## 8. 문구 (Copy / microcopy)

- 목소리는 **따뜻하고 아기자기하고 담백하게.** 다꾸 친구가 말하듯. 마케팅 과장·느낌표 남발·이모지 폭탄 금지.
- **금지어**: Coffee Passport / Passaporto / 여권, 별점, 팔로워, 리뷰 — §1 avoid-list의 언어를 UI에 되살리지 마라.
- 한국어/영어 혼용은 제품 톤에 맞게. 다꾸 도메인어(속지, 다꾸)는 자연스럽게 쓰되 UI 라벨은 짧고 명료하게.
- Moka(마스코트)는 **콘텐츠**로 등장하지, 시스템 목소리를 가로채지 않는다.

## 9. 모바일 검수 (Mobile QA — 필수)

Momenti는 **모바일 우선**이다. 새 UI는 아래 폭에서 **깨지지 않는지 반드시 확인**하라:

- **320px** (작은 폰) — 가로 스크롤·잘림·겹침 없어야 함
- **375px** (기본 폰) — 주 타깃
- **414px** (큰 폰)
- **768px** (태블릿/데스크톱 폭) — 하루 페이지 3:4 레터박스가 자연스럽게 중앙 정렬

각 폭에서: 하루 페이지 3:4 유지, 스티커/텍스트 배치·이동·삭제 동작, 툴바가 잘리지 않음, 안전 영역(노치) 침범 없음.

## 10. Momenti anti-slop (하지 말 것 모음)

"어디서 본 듯한 기본 웹 UI"로 흘러가는 것을 경계하라. 아래는 즉시 회귀 신호다:

- 부트스트랩/머티리얼 기본 룩, 파랑 primary 버튼, 딱딱한 카드 그림자
- 형광·고채도 SaaS 색, 순수 흑/회 텍스트
- 정보 밀도 높은 대시보드 레이아웃, 데이터 테이블 UI
- 장식용 통통 애니메이션, 스켈레톤 셔머, 시차 스크롤
- 별점·팔로워·리뷰·지도·화폐 UI(§1 avoid-list)
- 토큰 대신 하드코딩한 색/폰트/반경
- 요청받지 않은 리팩터·재설계·"김에 개선"

## 11. 예외 (Exceptions)

- 위 원칙과 다르게 가야 할 **명확한 이유**가 있으면, 조용히 어기지 말고 **사람에게 이유와 함께 제안**하라.
- `docs/product.md`의 잠금 결정(D1–D15)과 충돌하는 제안은 코드로 옮기기 전에 반드시 확인받는다.
- 프로토타입 단계 특성상 `App.jsx`가 토큰을 인라인 중복 보유하는 것은 **현재 허용된 예외**다(값은 토큰과 1:1). 새 값은 토큰에서 가져오되, 대규모 토큰-추출 리팩터는 별도 요청이 있을 때만.

## 12. 신규 UI 작업 체크리스트 (Definition of done)

새 UI/변경을 끝내기 전에 스스로 검수하라:

- [ ] `DESIGN.md`(이 문서)와 `docs/product.md` 범위를 먼저 읽었다.
- [ ] 색·폰트·반경·그림자를 **토큰**에서 가져왔다(하드코딩 없음).
- [ ] 새 요소는 `PageElement`/`StickerVisual` 등 **기존 시스템에 합류**했다(평행 시스템 없음).
- [ ] **320 / 375 / 414 / 768px**에서 레이아웃이 깨지지 않는다.
- [ ] 상호작용 응답 <100ms, 애니메이션 60fps(또는 단순화), **`prefers-reduced-motion` 존중**.
- [ ] press/active/focus 상태와 `aria-label`이 있고, 제거한 UI의 잔재(핸들러·라벨·주석)가 없다.
- [ ] avoid-list 언어/룩(§1·§10)을 되살리지 않았다.
- [ ] `npm run lint` · `npm run build` 통과, 저장 스키마·사용자 데이터 불변.
- [ ] 요청받지 않은 리팩터·재설계·기능을 추가하지 않았다. PR은 작게.
